## 第 4 章 缺陷清单

### BUG-01 超预算 Notification 未弹出（L2-08）

| 项目 | 内容 |
|------|------|
| 严重程度 | Medium |
| 关联 AC | AC-03-05 |
| 关联用例 | L2-08 |
| 预期 | 页面加载时，若本月支出超预算，右上角自动弹出 ElNotification（标题"超预算提醒"，消息"本月支出已超预算！"，红色/error 样式，5 秒后消失） |
| 实际 | 预算进度卡片正确显示红色进度条和 120%，但 Notification 弹窗在两次测试中均未被观察到（含 Cmd+R 刷新后立即观察） |
| 代码定位 | `src/App.vue` 第 201-216 行。代码存在，使用 `watch({ immediate: true })` 监听 `budgetProgress.value.isOverBudget`。可能原因：(1) Notification 在 Vue 初始化阶段触发但 5s 后消失，mano-cua 截图时已过期；(2) 响应式链路问题导致 `isOverBudget` 在 watch 初次执行时尚未为 `true` |
| 建议 | 1. 检查 `getBudgetProgress()` 在组件挂载时的返回值时序 2. 考虑将 duration 延长或改为手动关闭 3. 用浏览器 DevTools 断点确认 watch 是否实际触发 |

## 第 5 章 观察项（非阻塞）

### OBS-01 金额超限依赖浏览器原生截断（L2-01 场景 D）

输入 10,000,000 时，浏览器 `<input type="number" max="9999999.99">` 在 blur 事件自动截断为 9999999.99，应用层未显示自定义错误提示。用户可能不知道金额被静默修改。

**建议：** 添加应用层 blur 校验，当值超过上限时显示"金额不能超过 9,999,999.99"提示。

### OBS-02 防重复提交机制为表单重置（L3-07）

PRD 描述为"按钮 disable 1 秒"，实际实现为"提交成功后立即清空表单"。功能效果等价（快速连击不会产生重复记录），但实现方式与 PRD 描述不一致。

**建议：** 与 PM 确认是否需要补充 button loading/disabled 状态反馈（更符合用户预期的交互模式）。
